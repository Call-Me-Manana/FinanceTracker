# Finance Tracker

Android-приложение для учета личных финансов: доходы, расходы, категории, цели накоплений, бюджеты и базовая аналитика. Проект сделан как учебный трекер, но с подходами, которые используются в реальной Android-разработке: Jetpack Compose, Room, ViewModel, Flow и разделение UI, данных и бизнес-логики.

## Что умеет приложение

- Добавление, редактирование и удаление транзакций
- Быстрое добавление транзакции с главного экрана через FAB
- Группировка операций по дням
- Свайпы для действий с транзакциями
- Категории доходов и расходов с иконками
- Редактирование категорий
- Месячные бюджеты для расходных категорий
- Прогресс бюджета по категориям за текущий месяц
- Раздел финансовых целей
- Создание, редактирование, пополнение и удаление целей
- Прогресс накопления по каждой цели
- Аналитика по доходам, расходам и категориям
- Фильтр транзакций по периоду: сегодня, неделя, месяц, все
- Поиск по транзакциям и категориям
- Краткая статистика по выбранному периоду

## Архитектура

Проект постепенно развивается в сторону простой MVVM-структуры:

```text
UI Screen
   ↓
ViewModel
   ↓
Repository
   ↓
DAO
   ↓
Room Database
```

Основные слои:

- `data` — Room-сущности, DAO и модели данных
- `repository` — слой между ViewModel и DAO
- `ui` — Compose-экраны и компоненты
- `utils` — вспомогательные функции для дат, денег и расчетов

## Основные разделы

### Главный экран

На главном экране отображаются баланс, доходы, расходы, список транзакций, фильтр периода, поиск и короткая статистика.

Транзакции можно добавлять прямо с главного экрана через плавающую кнопку. Редактирование работает через тот же диалог: если транзакция не передана, создается новая; если передана существующая, она обновляется через `copy`.

### Категории и бюджеты

Категории делятся на доходные и расходные. Для расходных категорий можно указать месячный бюджет. Приложение считает траты за текущий месяц и показывает прогресс.

Если бюджет превышен, значение подсвечивается цветом ошибки.

### Цели

Раздел целей помогает отслеживать накопления. Для каждой цели можно указать сумму, текущий прогресс, пополнять ее и редактировать.

### Аналитика

В аналитике используются графики и агрегированные данные по транзакциям. Это помогает видеть структуру расходов и динамику финансов.

## Запуск проекта

1. Открой проект в Android Studio.
2. Дождись синхронизации Gradle.
3. Запусти приложение на эмуляторе или физическом устройстве.

Или через терминал:

```bash
./gradlew assembleDebug
```

## Учебные темы, которые покрывает проект

- Декларативный UI на Jetpack Compose
- Состояние в Compose: `remember`, `mutableStateOf`, `collectAsState`
- Работа с базой данных через Room
- Связи между транзакциями и категориями
- Асинхронные операции через coroutines
- Потоки данных через Flow и StateFlow
- Навигация между экранами
- Диалоги добавления и редактирования
- Фильтрация и поиск данных

- Работа с датами и периодами
